# 关键发现汇总

## H 方案 (阶梯锁利) — 首个打败 baseline 的方案

3 年 BTC 1h 回测: **+718% (比 baseline +434% 多 65%)**

### 规则
- 入场 / 止损 / 信号检测: **跟 baseline 完全一致** (突破确认 + F6 regime + T5 容差 + 2% buffer)
- 止盈目标: **8R** (vs baseline 2R)
- **阶梯锁利**:
  - 价格 → +2R: SL 上移到 +1R (保 1R)
  - 价格 → +4R: SL 上移到 +2R (保 2R)
  - 继续到 +8R 全身而退, 或被升高 SL 拦下

### 数据对比

| 维度 | baseline 2R | H (8R 阶梯) |
|------|------------|------------|
| 胜率 | 47.1% | **47.1%** (相同) |

| 总收益 | +434% | **+718%** |
| 最大回撤 | -47% | -78% |
| 平均 R/单 | +0.37R | **+0.61R** |

### 为啥能赢
Baseline 在 +2R 就关, 错过 BTC 1h 反转启动后常见的 +5R/+6R/+8R 大行情。
H 让赢家继续跑, 但用阶梯锁保护已赚部分。

## 第三个突破: 连亏冷却机制 + H 阶梯锁 = 新王者

**baseline → +478%, 新王 → +840%, 全维度碾压**

### 终极组合: H + 连亏 2 停 24h
- 信号检测 / 入场 / 初始 SL: 跟 baseline 一致
- TP 目标: 8R
- **H 阶梯锁**: 2R 锁 +1R, 4R 锁 +2R
- **冷却**: 连续 2 次 SL hit 后, 暂停 24 小时不开新单

### 数据对比
| 维度 | baseline | **H + 连亏 2 停 24h** |
|------|---------|---------------------|
| 总收益 | +478% | **+840%** (+76%) |
| 胜率 | 47.0% | **55.2%** (+8.2 个点) |
| 最大回撤 | -17R | **-11R** (-35%) |
| 最大连败 | 15 单 | **11 单** (-27%) |
| 平均 R/单 | +0.41 | **+1.02** |
| 笔数 | 449 | 317 |

### 守势王: 连亏 2 停 24h + 单向冷却
- 单向冷却 = 多单连亏只暂停多单, 空单照常 (反之亦然)
- 回撤 **-6R** (baseline -17, 减 65%), 最大连败 **6 单**
- 总收益 +655%, 笔数 366

### 性价比王: H + 连亏 2 停 24h + 仅多
- 平均每单 **+1.41R** (各方案最高)
- 胜率 58.7%, 196 笔, +718%

### 部署优先级
1. **#1 推荐: H + 连亏 2 停 24h** — 全维度最优
2. **#2 推荐 (心态稳): 连亏 2 停 24h + 单向冷却** — 回撤最浅
3. **#3 推荐 (牛市): H + 连亏 2 停 24h + 仅多** — 单笔效率最高

### 代码改动 (部署任一方案)
1. `state.json` 加 `pause_until_ts` 字段, 默认 0
2. `signal_bot.py` 的 `detect_new_signals`:
   ```python
   if now_ts < state.get("pause_until_ts", 0):
       return []  # 冷却中, 跳过本次扫描
   ```
3. 每次 SL hit 后, 检查最后 N 笔是否都是负, 是则:
   ```python
   state["pause_until_ts"] = exit_ts + 24 * 3600
   ```
4. 如果用 H 方案: `update_signal_status()` 加阶梯 SL 移动 (触及 +2R 时 SL→+1R, 触及 +4R 时 SL→+2R)
5. TP 目标改 `r_multiple=8.0` (从 2.0)
6. TG 推送加"冷却中"通知

## 测试过、已确认不如 baseline 的方向 (避免重复)

- 缩小 SL buffer (0.3% ~ 1.5%) — 全部败
- 拉长 TP (3R~5R 单纯拉远, 无 trailing) — 总收益小升但回撤翻倍
- Fib 锚定 SL/TP (用 swing_high 为基准) — 数学陷阱, 实战亏
- 分批止盈 (50% @1R + 50% @2R + 各种 SL 移动) — 都败
- EMA 换成 4h / 1D 时间框架 — 都败
- EMA 周期换 50/100/300/500/1000 — 200 最优
- 单独用 RSI / CCI 过滤 — 笔数太少或胜率太低
- 单独用 MACD / Supertrend / DMI / HMA 替代 EMA-200 — 全部败
- ATR 波动率过滤 (任何门槛) — 都没改善
- BE 保本类 trailing — 太敏感, 反弹被洗掉
